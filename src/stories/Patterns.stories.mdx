<h2 className="ds-heading2"> Notifying the user of data collection and asking them to consent to it</h2>

<div className="ds-flex">
<div>
  <p>
  We have a responsibility to ask users for consent when we want to collect, store, or use their data especially personal information.
  </p>
  <p className="ds-body-bold">
  This pattern asks the user to:
  </p>
  <ul>
    <li>read a summarized privacy notice</li>
    <li>confirm that they clearly understand and freely agree to how their data will be collected and used</li>
  </ul>

<h2 className="ds-heading2"> Notifying the user of data collection and asking them to consent to it</h2>
<p className="ds-body-bold">
Use this pattern when:
</p>
<ul>
 <li>privacy notices (statements, terms of use, etc.) are required. For example, signing up for a new service or storing data in cookies</li>
 <li>you are required by law to record when and how consent is given. This includes the version of a privacy notice the user agreed to</li>
 <li>the user understands what data is collected, where it is stored, what it is used for, and for how long</li>
 <li>the user is able to opt-out, correct, or delete their data at any time</li>
</ul>

<h2 className="ds-heading2">When not to use this pattern</h2>
<p className="ds-body-bold">
Don’t use this pattern when:
</p>
<ul>
  <li>the type of data collected, where it is stored, what it is used for, and for how long, is unspecified or will change often</li>
  <li>the user cannot see or access the other data that organization collects and its purposes or uses</li>
  <li>there isn’t a clear process for the user to opt-out, correct, or delete their data at any time</li>
</ul>

<h2 className="ds-heading2">How this pattern works</h2>
<p>
When you collect a user’s data, make sure the following are <b>present, easy to find, read, and use.</b>
</p>

<h4 className="ds-heading4">Privacy statement</h4>
<p>
A summarized privacy statement highlighting the type(s) of data collection, storage, and use, with links to the full statement. Be sure to use proper heading levels, typically a h2 is used.
</p>

<h4 className="ds-heading4">Opt-out Description</h4>
<p>
A brief description of how to opt-out, correct, or delete their data later. The section should start with the same level heading as the Privacy statement, typically a h2.
</p>

<h4 className="ds-heading4">Example of a privacy statement</h4>
<div className="ds-rounded ds-shadow-md ds-pt-20px ds-pb-20px ds-px-24px">
    <div>
      <h2 className="ds-heading2">
        How to unsubscribe from [identify what the user is unsubscribing from]
      </h2>
      <p className="ds-body">
        If you have previously signed up to become a participant and no longer
        wish to be contacted for future [explain in detail what the user is
        unsubscribing from], you can{" "}
        <Link href="#unsubscribe" id="link" text="unsubscribe" /> yourself from
        the participant list and we will remove your personal information.
      </p>
    </div>
</div>

<h4 className="ds-heading4">Affirmative action</h4>
<p>
An affirmative action the user can take to opt-in by agreeing to the privacy statement (for example, a checkbox that isn’t pre-selected).
</p>

            18 years old, or older. I understand that I can withdraw from this
            [service name] at any time without consequence.
